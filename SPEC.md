## 機能名
FormGuide — 外国語入力フォーム視覚ガイド（MVP）

## ユーザーストーリー
外国の求人サイトや開発ツールの設定画面で「何を入力するか」は分かっているが「どこに入力すべきか」が分からないユーザーとして、スクリーンショットと入力したい内容をテキストで送ると、どの欄に何を入れればよいかを矢印・番号・ハイライトで可視化した注釈画像と日本語の説明文を受け取りたい。なぜなら、従来のAIチャットでは「ここに入力してください」と文字で言われても、外国語UIの中でその「ここ」を特定できないことが多いからだ。

## 受け入れ基準
- [ ] ユーザーはスクリーンショットを①ファイル選択ダイアログからのアップロード、②ドラッグ＆ドロップ、③Ctrl/Cmd+V によるクリップボード画像貼付の 3 経路で投入できる
- [ ] ユーザーは「入力したい内容」を自由記述テキスト（複数行可）で添えて送信できる
- [ ] 送信後、サーバーは GPT-4o Vision にスクショとテキストを渡し、各入力欄に対して `{ label: 該当欄の説明, value: 入れるべき値, bbox: {x,y,w,h}（画像座標・0-1 正規化）, note: 補足説明 }` を含む構造化 JSON を取得する
- [ ] クライアントは元画像の上に SVG オーバーレイで矢印・番号バッジ・半透明ハイライト枠を重ねて注釈画像を描画する（元画像は一切改変しない）
- [ ] 注釈の右側（または下部）に各番号に対応する日本語の説明リストが表示される（「①Name欄 → 山田太郎（姓名まとめて入力）」等）
- [ ] ユーザーはチャット形式で追加質問ができ、同一セッション内では直前の画像と解析結果を文脈として保持する
- [ ] パスワード、クレジットカード番号、マイナンバー、CVV、銀行口座番号などの機密情報を入力テキスト中に検出した場合、**処理を完全拒否**し、どの項目が検出されたかを警告として表示する
- [ ] 画像はサーバーに永続保存されず、レスポンス送信完了後にメモリ/一時領域から破棄される
- [ ] OpenAI API キーは `OPENAI_API_KEY` としてサーバーサイドのみで使用され、`NEXT_PUBLIC_` プレフィックスには含まれない
- [ ] TypeScript strict mode・ESLint・`npm run build` がすべてエラーなしで通る

## エッジケース
- **画像サイズが極端に大きい（> 10MB / > 4096px）**: クライアント側で自動リサイズして API 送信上限（GPT-4o 推奨解像度）に収める。リサイズ倍率をサーバーに伝え bbox の座標系は常に「リサイズ後の画像」に正規化
- **GPT-4o が入力欄を特定できない**: `bbox` が返らなかった項目は注釈オーバーレイに載せず、説明リスト末尾に「未特定: この項目は画像から該当欄を特定できませんでした」として表示
- **スクショに入力フォームが存在しない（風景写真等）**: エラーレスポンスで「フォーム要素が検出されませんでした」と返し、再アップロードを促す
- **入力テキストに機密情報が含まれる**: 正規表現＋キーワード辞書（password, credit card, SSN, マイナンバー, CVV, PIN 等）で事前検出 → 検出時は OpenAI に送信せず、どの項目で検出したかを明示して 400 エラー相当のレスポンスを返す
- **入力テキストが空 or 画像のみ**: 400 エラーで「入力したい内容を記述してください」と返す
- **スクショ中に複数の同種入力欄がある（例: 配送先と請求先の住所）**: GPT-4o に文脈から判断させ、曖昧な場合は説明文に「※配送先の住所欄と判断しました」のように判断根拠を付す
- **チャットでユーザーが新しい画像を貼り付けた**: 直前の画像コンテキストを破棄し、新しい画像でやり直す（混在を避ける）
- **OpenAI API がレート制限 or タイムアウト**: 30 秒でタイムアウト。ユーザーには「AI が混雑しています。しばらくしてから再試行してください」と表示
- **bbox 座標が画像範囲外を返す**: クライアント側で 0-1 にクランプし、範囲外なら該当注釈を描画せず説明リストのみに載せる

## 技術的制約
- **フレームワーク**: Next.js 14 App Router / TypeScript strict mode（既存 CLAUDE.md 規約準拠）
- **スタイル**: Tailwind CSS + Framer Motion（アニメーション）
- **状態管理**: Zustand（チャット履歴・現在画像・解析結果）
- **アイコン**: Lucide React
- **AI**: OpenAI API の GPT-4o Vision（`gpt-4o` モデル、JSON mode / structured outputs）をサーバーサイド Route Handler (`app/api/analyze/route.ts`) 経由で呼び出す
- **注釈描画**: クライアントサイド SVG オーバーレイ。元画像は `<img>`、その上に `<svg>` を absolute で重ね、`viewBox` を元画像のピクセル解像度に合わせて座標変換する。サーバー側で画像を再生成しない（Sharp/Canvas 不要）
- **画像入力経路**: File input / Drag & Drop / ClipboardEvent (`paste`) の 3 系統。いずれも `File` → `FileReader.readAsDataURL` → base64 data URL 化してメモリ上のみで保持
- **セッション**: サーバー側はステートレス。チャット文脈はクライアント側 Zustand に保持し、API リクエスト毎に必要な履歴のみサーバーに送る（画像は直前のものを base64 で再送）
- **機密情報検出**: サーバー側 Route Handler 冒頭で `lib/security/sensitive-detector.ts` を実行。正規表現（クレカ Luhn 簡易版、マイナンバー 12 桁、SSN 9 桁）＋多言語キーワード辞書（ja/en）
- **テスト**: Vitest（ユーティリティ・機密検出・React コンポーネント）＋ Playwright（貼付→解析→描画の E2E 1 本）
- **デプロイ**: Vercel。`OPENAI_API_KEY` は Vercel 環境変数で設定
- **禁止事項**: `any` 型禁止、`console.log` 禁止、画像のサーバー永続化禁止、APIキーのクライアント露出禁止、class コンポーネント禁止

## 非スコープ（今回やらないこと）
- ユーザー情報テンプレートの事前登録（住所・氏名等のプリセット）
- ブラウザ拡張による Web ページへの直接注釈オーバーレイ / 自動入力
- 日本語以外への多言語対応（UI 言語・説明文言語は日本語固定）
- ユーザー認証・アカウント機能・利用履歴の保存
- 画像 URL 直接指定による取得（CORS / SSRF / 認証必須サイトへの配慮コストが高いため MVP では見送り）
- GPT-4o 画像生成 API による注釈焼き込み（クライアント SVG オーバーレイで代替）
- 複数スクリーンショット同時解析（1 リクエスト＝ 1 画像）
- 注釈付き画像のダウンロード / 共有機能（将来検討）
- レート制限・課金・ユーザー別クォータ（MVP は個人用途想定）
- PWA / オフライン対応
